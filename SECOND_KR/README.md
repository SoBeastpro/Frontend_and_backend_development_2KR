# 🛍️ Shop Application

Полофункциональное веб-приложение с системой аутентификации, JWT токенами, RBAC (контролем доступа на основе ролей) и управлением товарами.

## 📋 Содержание проекта

Проект состоит из двух частей:
- **Backend**: Node.js + Express (API серверный)
- **Frontend**: React (клиентская часть)

## 🚀 Быстрый старт

### Требования
- Node.js (v14 или выше)
- npm или yarn

### Установка и запуск Backend

```bash
cd backend
npm install
npm start
```

Сервер будет запущен на `http://localhost:3000`

### Установка и запуск Frontend

```bash
cd frontend
npm install
npm start
```

Приложение откроется в браузере на `http://localhost:3000` (frontend использует port 3000, backend запущен на отдельном port)

## 📚 Функциональность

### Аутентификация и Авторизация
- ✅ Регистрация новых пользователей
- ✅ Вход в систему с email и паролем

- ✅ Просмотр списка всех товаров
- ✅ Просмотр деталей товара
- ✅ Создание новых товаров (для sellers и admin)
- ✅ Редактирование товаров (для sellers и admin)
- ✅ Удаление товаров (только для admin)

### Управление пользователями (Admin)
- ✅ Просмотр списка всех пользователей
- ✅ Просмотр информации о пользователе

## 🔐 Аутентификация

### Процесс входа
1. Пользователь вводит email и пароль
2. Backend проверяет учетные данные
3. Сервер выдает access токен и refresh токен
4. Токены сохраняются в localStorage
5. Все последующие запросы содержат access токен в заголовке `Authorization: Bearer <token>`

### Обновление токенов
- Access токен имеет время жизни 15 минут
- Refresh токен имеет время жизни 7 дней
- При истечении access токена frontend автоматически запрашивает новый через refresh токен

## 📊 API эндпоинты

### Аутентификация
```
POST   /api/auth/register     - Регистрация пользователя
POST   /api/auth/login        - Вход в систему
POST   /api/auth/refresh      - Обновление токенов
GET    /api/auth/me           - Получить текущего пользователя
```

### Товары
```
GET    /api/products          - Получить список товаров
GET    /api/products/:id      - Получить товар по ID
POST   /api/products          - Создать товар (seller, admin)
PUT    /api/products/:id      - Обновить товар (seller, admin)
DELETE /api/products/:id      - Удалить товар (admin)
```

### Пользователи
```
GET    /api/users             - Получить список пользователей (admin)
GET    /api/users/:id         - Получить пользователя по ID (admin)
PUT    /api/users/:id         - Обновить пользователя (admin)
DELETE /api/users/:id         - Заблокировать пользователя (admin)
```

## 🔧 Использованные технологии

### Backend
- **Express.js** - веб-фреймворк для Node.js
- **bcrypt** - хеширование паролей
- **jsonwebtoken** - создание и проверка JWT токенов
- **nanoid** - генерация уникальных ID
- **cors** - поддержка кросс-доменных запросов

### Frontend
- **React** - библиотека для создания UI
- **React Router** - навигация между страницами
- **Axios** - HTTP клиент для API запросов
- **localStorage** - сохранение токенов на клиенте

## 📝 Примеры использования

### Создание учетной записи
1. Нажмите на кнопку "Регистрация"
2. Введите email, имя, фамилию и пароль
3. Нажмите "Зарегистрироваться"

### Вход в систему
1. Нажмите на кнопку "Вход"
2. Введите email и пароль
3. Нажмите "Войти"

### Создание товара (для продавцов)
1. После входа нажмите "➕ Добавить товар"
2. Заполните форму с информацией о товаре
3. Нажмите "Создать товар"

### Управление пользователями (для администратора)
1. После входа как администратор, нажмите "👥 Пользователи"
2. Смотрите список всех пользователей
3. Редактируйте информацию или блокируйте пользователей

## 🐛 Известные ограничения

- Данные хранятся в памяти и теряются при перезагрузке сервера
- Нет персистентной базы данных (используется массив в памяти)
- Аутентификация использует простой JWT (без дополнительной защиты)
